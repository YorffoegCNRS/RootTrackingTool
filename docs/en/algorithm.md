# RTT : Algorithm

* 🇫🇷 [Version française](../fr/algorithm.md)


## Architecture

RTT is organized into two main modules, corresponding to two complementary processing steps: (i) segmentation/cleaning of masks generated by Ilastik, followed by (ii) advanced analysis of the root network through skeletonization and graph analysis.

### 1\. Segmentation and preparation of masks (Part I)

**Entries :** pre-segmented images (root/leaf/background color codes).
**Outputs :** cleaned binary masks and exported image series (roots and/or leaves) + tracking graphs

The main workflow is as follows:
* Selection of the dataset folder + output folder
* Definition of areas of interest (roots and leaves)
* Cropping of areas of interest
* Segmentation by RGB thresholding, with post-processing options:
  * Retention of the largest connected component
  * Filtering by minimum area, with “recovery” of small nearby objects via distance to the centroid
  * Removal of small objects (min object size)
  * Morphological closing (size + shape of the kernel)
  * Updating of a preview with global application
* Export of masks and display of evolution graphs (size of roots/leaves and convex hull)



### 2\. Root architecture analysis (Partie II)

This part transforms the root binary mask into a topological representation (skeleton + graph), then calculates global and local morphometric metrics.

Conceptual pipeline:

```
Binary mask
      │
      ▼
Morphological cleaning
      │
      ▼
Skeletonization
      │
      ▼
Graph construction
      │
      ▼
Extracting the main path
      │
      ▼
Extraction of secondary branches
      │
      ▼
Metrics calculation
```

#### a. Pre-treatment of the mask

* Optional resizing if the image exceeds a maximum size (to limit computational cost)
* Optional local temporal fusion (continuity between days), with closure (parameter *fusion tolerance pixel* for kernel size) applied before an **AND** operation between masks
* Morphological cleaning: closure (*parameter closing radius* for kernel size) + removal of small objects (parameter *minimum object size*)
* Optional connection of nearby fragments (maximum distance with the *maximum connection distance* parameter + connection thickness via the *connection thickness* parameter)
* Optional calibration (*pixels/cm* parameter): generation of metrics in physical units
* Virtual grid (X rows × Y columns): calculation of root lengths per cell + visualization in a heatmap


#### b. Skeletonization and graph construction

* Extraction of a morphological skeleton with a thickness of 1 pixel, preserving the topology
* Construction of a graph (8-connectivity)
  * Nodes: skeleton pixels (y, x coordinates)
  * Edges: 8-connected connection, weighted by discrete Euclidean distance.
  * Weight:
    - horizontal/vertical neighbor: 1
    - diagonal neighbor: √2
  * Degree of a node: number of neighboring nodes
  * Detection of endpoints (degree 1 nodes) and junctions (degree ≥ 3 nodes)


#### c. Main root detection and temporal analysis

* Day 1: Selection of a starting point (highest end) and end point (lowest end), then shortest weighted path calculated using Dijkstra's algorithm on an undirected graph.
* Subsequent days: use of the previous day's path as a reference, via a continuity bias (*main path bias* parameter) applied to the edge weights, in order to stabilize detection over time (see the dedicated section below for more details).
* Persistence mechanism: if no path is found, the previous day's path is retained.
* Extension of the main path upwards and downwards until plausible endpoints are reached (process of these extensions below).


**Temporal continuity bias :**
In order to stabilize the detection of the main root between two days, a bias is applied to the weights of the graph edges.
* For each node, the minimum Euclidean distance between that node and the set of pixels constituting the main path of the previous day is calculated.
* For each edge (u, v), we define: `d̄ = (d(u) + d(v)) / 2`
* The weight of the edge becomes `w_biased = w_base × (1 + α × (d̄ / d_max)^2)` where
- w_base ∈ {1, √2} (8-connectivity),
  - d_max is the maximum distance observed across all nodes in the graph,
  - α = 2 × main_path_bias
* The shortest path is then calculated using Dijkstra's algorithm on these biased weights.
* This mechanism introduces continuous spatial regularization, limiting abrupt topological jumps between consecutive days.

**Upward extension :**
* From the first point on the path (high anchor)
* Unidirectional BFS: explores all accessible nodes without ever passing through nodes already on the main path. This BFS is constrained to the subgraph obtained after temporarily removing the edges belonging to the main path.
* Among all the endpoints reached, selects the one with the minimum Y coordinate (the highest)
* Traces the unique path from the anchor to this endpoint via reconstruction of the parent dictionary
* Guarantees that it is impossible to create a double branch or a cycle

**Downward extension:**
* From the last point on the path (lower anchor)
* Unidirectional BFS: explores all accessible nodes without passing through nodes already on the path (except the anchor itself). This BFS is constrained to the subgraph obtained after temporarily removing the edges belonging to the main path.
* Among all the endpoints reached, selects the one with the maximum Y coordinate (the lowest)
* Traces the unique path from the anchor to this endpoint
* Handles lateral deviations at the end of the root: even if the root branches left or right at the bottom, the algorithm finds the lowest reachable endpoint

This strategy ensures that:
* The main path is always simple (no loops, no double branches).
* The extension reaches the true global endpoints even in the case of multiple bifurcations.
* The result is reproducible (no ambiguous choices at bifurcations).


#### d. Extracting secondary roots

After removing nodes belonging to the main path, each remaining connected component is considered a potential secondary root.

**Process:**
1. Remove the main path from the graph
2. Detect remaining connected components (NetworkX)
3. For each component: calculation of the longest path
4. Optional pruning of short endings (< min_branch_length)

Lengths are calculated by traversing the skeleton pixel by pixel to ensure maximum accuracy (exact_skeleton_length).


### Algorithmic complexity

Let N be the number of pixels in the skeleton, E the number of edges in the graph (E ≈ cN (with c ≈ 4))

**Construction and preprocessing:**
- Graph construction: O(N)
- Endpoint detection: O(N)
- Calculation of connected components: O(N)

**Main root detection:**
- *With time reference (following days)*:
  - Calculation of biased weights: O(E) = O(N)
  - Dijkstra with bias: O(E log N) = O(N log N)
- *Without reference (first day or disconnected graph)*:
  - Sorting components: O(C log C) where C = number of components
  - Dijkstra per component: O(N_i log N_i), N_i = size of component i

**Path extension (strict BFS):**
- BFS upwards: O(N)
- BFS downwards: O(N)
- Path reconstruction: O(L) where L = extension length

**Secondary root extraction:**
- Residual component detection: O(N)
- Length calculation: O(N)

**Total complexity: O(N log N)** (dominated by Dijkstra but remains compatible with graphs containing several hundred thousand skeletal pixels).


### Model assumptions

**Spatial orientation:**
- The main root is generally oriented vertically (dominant Y axis).
- The image coordinate system places Y=0 at the top.
- The main root connects the highest point (Y min) to the lowest point (Y max).

**Temporal continuity:**
- The main root maintains a similar topology between consecutive days.
- Spatial deviations are limited and gradual.
- A continuity bias (main_path_bias = 20 by default, or α = 40) forces the path to follow the trajectory of the previous day.

**Data quality:**
- Masks are correctly segmented (roots = white pixels).
- Skeletonization preserves the topology of the root network.
- Connected components correspond to real biological structures.

**Robustness to edge cases:**
- **Disconnected graphs**: selection of the component with the greatest vertical amplitude, with vertical amplitude (max(Y) − min(Y)).
- **Multiple bifurcations**: BFS guarantees selection of the most extreme endpoint.
- **Missing days**: persistence of the time reference from the last successful day.
- **Lateral deviations**: BFS explores all directions to reach the global endpoint.
